Mark McCorkle - Progressive Enhancement

Progressive Enhancement is a strategic design philosophy that focuses on delivering content first followed subsequently with additional advanced features (subject to the users’ available resources), thereby giving the end user a universal basic site content access and functionality. The Progressive Enhancement methodology used in Web Development curriculum teaches primarily HTML (content sections first), then moves to CSS (presentation layer), and finally the scripting-side. The modularization of web development has many advantages over the traditional teachings of web development of the past which includes accessibility, performance, cross-browser and cross-platform support among many others. This philosophy allows for the focusing of the most important part, the content. It involves separating the content from the presentation and server-side layers so all users can easily obtain the site content. By keeping these layers separate it allows the developer to maintain site structure/organization thereby making the site flow properly for the developer, but most importantly the end user. This grants the users’ system and capabilities to not limit access to the content but limit the presentation layers abilities as it relates to the user’s hardware. This gives everyone access to the most important part of the site with basic functionality and leaves the rest as icing on the cake. Web developing requires that developer’s acknowledgement that they create for a wide variety of users, systems, platforms, bandwidth, and geological based limitations. With Progressive Enhancement, the developer can structure the sites contents to match the needs of each individual (if so desired) creating a rich and robust experienced tailored to the desired audience.
